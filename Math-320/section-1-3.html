Section 1.3 Guided Practice 1.3: Vector Equations

Overview.

In Section 1.1, we introduced a new mathematical object, matrices, to help study linear systems of equations. In this section, we introduce vectors, another possibly more familiar mathematical object that will aid us.

After describing vectors and their basic properties both algebraically and geometrically, the section begins relating them back to linear systems by introducing the very important concepts of linear combinations and spans. These two highly related concepts will be used repeatedly throughout the rest of the semester. Example 5 and the following discussion illustrate the strong connection between questions about linear combinations and questions about linear systems. The geometric descriptions should be used to help build intuition about these new concepts.

Subsection 1.3.1 Basic Learning Objectives

You should be able to perform these tasks with reasonable fluency when you arrive to class.

  • Compute sums and scalar multiples of column vectors in \(\mathbb{R}^n\text{.}\)
  • List examples of linear combinations of a given set of vectors, specifying weights.
  • Translate among vector equations, linear systems, and augmented matrices.
  • Define the span of a set of vectors and produce elements of the span of a given set of vectors.

Subsection 1.3.2 Advanced Learning Objectives

In addition to mastering the basic objectives, here are the tasks you should be able to perform after class, with practice:

  • Geometrically describe spans of sets of vectors in \(\mathbb{R}^2\) or \(\mathbb{R}^3\text{.}\)
  • Determine if a given vector is in the span of a set of vectors.
  • Write a given vector as a linear combination of a set of other vectors, if possible.

Subsection 1.3.4 Sage Commands

When written as column vectors, vectors behave like \(n \times 1\) matrices, and they can be coded as such in Sage. As a shortcut, you can specify the number of rows (number of entries in the vector) and then include a single list of the entries.

However, we often want to treat vectors as vectors specifically. In Sage, we use the vector command, which is similar to the matrix command. We do not need to state the number of entries with this command; we simply list them out. Note also that Sage displays the result in row form (an alternative display form discussed in the textbook), but they are treated the same as column vectors.
